Arithmetic Reference

A high-precision mathematical engine implemented in C and x86 assembly, exposed as a Discord bot. All computations operate on 2048-bit extended-precision floating-point values, giving approximately 617 significant decimal digits.

C99 x86 Assembly 2048-bit Precision CORDIC Series Gaussian Quadrature Expression Trees
§ 01

Precision Model

2048 Mantissa bits
~617 Decimal digits
32 Exponent bits
256 64-bit limbs

The internal representation uses a 256-limb array of unsigned 64-bit integers (little-endian) for the mantissa, with a separate 32-bit signed exponent and a sign bit. Arithmetic inner loops are hand-written x86-64 assembly using ADC/SBB carry chains across all 256 limbs.

ℹ All results are displayed to a configurable number of significant figures (default 40). Use the /prec sub-command to increase or decrease the displayed precision up to 617 digits.
§ 02

Invocation

All commands are prefixed with /calc followed by a sub-command. Arguments may be integers, decimals, or expressions in standard infix notation. Very large or very small values may be written in scientific notation: 1.5e300.

syntax/calc <command> <args…> # Examples: /calc add 3.14159 2.71828 /calc pow 2 1024 /calc zeta 3 /calc mat_inv "[[1,2],[3,4]]"
⚠ Matrix and polynomial arguments must be enclosed in double quotes. Matrices use row-major nested bracket notation: "[[a,b],[c,d]]". Polynomials are given as coefficient lists from highest to lowest degree: "[1,0,-1]" represents x²−1.
§ 03

Basic Arithmetic

/calc add a b
Addition a + b
▶
Returns the sum of two 2048-bit values. Handles signed operands and subnormals correctly.
example
you ▸ /calc add 1.000000000000000000000000000001 2.999999999999999999999999999999 bot ▸ 4.000000000000000000000000000000
/calc sub a b
Subtraction a − b
▶
Subtracts b from a with full 2048-bit precision.
example
you ▸ /calc sub 1e200 9.99e199 bot ▸ 1e197
/calc mul a b
Multiplication a × b
▶
Multiplies two values using a schoolbook O(n²) limb-by-limb multiply with carry propagation in assembly. A Karatsuba fast path is used for large operands.
example
you ▸ /calc mul 123456789012345678901234567890 987654321098765432109876543210 bot ▸ 121932631137021795226185032733622923332237463801111263526900
/calc div a b
Division a ÷ b
▶
Long division using Newton–Raphson reciprocal refinement, producing a correctly rounded 2048-bit quotient.
example
you ▸ /calc div 1 7 bot ▸ 0.142857142857142857142857142857... (617 digits)
/calc pow base exp
Exponentiation baseᵉˣᵖ
▶
Supports integer exponents (binary exponentiation), rational exponents (via exp(exp·ln(base))), and large tower exponents. Negative bases with non-integer exponents return the principal complex value in rectangular form.
example
you ▸ /calc pow 2 1024 bot ▸ 17976931348623159077293051907890247336179769789423065727343...
/calc nroot x n
n-th root ⁿ√x
▶
Computes the principal real n-th root of x. Equivalent to pow(x, 1/n) but numerically more stable for large n. Odd roots of negative numbers are supported.
example
you ▸ /calc nroot 2 2 # √2 bot ▸ 1.41421356237309504880168872420969807856967187537694...
§ 04

Modular Arithmetic

/calc mod a m
Remainder a mod m
▶
Returns a mod m. Result is always non-negative (floored division semantics).
example
you ▸ /calc mod -17 5 bot ▸ 3
/calc powmod base exp m
Modular exponentiation
▶
Computes base^exp mod m efficiently using Montgomery multiplication. Handles 2048-bit moduli. Essential for RSA-scale computations.
example
you ▸ /calc powmod 2 1000000007 1000000009 bot ▸ 820981074
/calc modinv a m
Modular inverse a⁻¹ mod m
▶
Returns x such that a·x ≡ 1 (mod m) using the extended Euclidean algorithm. Returns an error if gcd(a, m) ≠ 1.
example
you ▸ /calc modinv 3 11 bot ▸ 4 # 3×4 = 12 ≡ 1 (mod 11)
§ 05

GCD / LCM

/calc gcd a b [c…]
Greatest common divisor
▶
Binary GCD algorithm (Stein's algorithm) implemented in assembly for efficiency. Accepts 2 or more integer arguments; folds left.
example
you ▸ /calc gcd 123456789 987654321 111111111 bot ▸ 9
/calc lcm a b [c…]
Least common multiple
▶
Computed as lcm(a,b) = |a·b| / gcd(a,b). Intermediate products are kept at full 2048-bit precision to avoid overflow.
example
you ▸ /calc lcm 12 18 24 bot ▸ 72
§ 06

Integer Factorisation

/calc factor n
Prime factorisation
▶
Factors n into primes using trial division for small factors, followed by Pollard's rho algorithm with Brent's cycle detection for larger composites. Miller–Rabin primality testing (40 rounds) confirms each factor.
example
you ▸ /calc factor 2147483647 bot ▸ 2147483647 = 2147483647 (Mersenne prime M31) you ▸ /calc factor 720720 bot ▸ 720720 = 2⁴ × 3² × 5 × 7 × 11 × 13
⚠ Factorisation of numbers with factors larger than ~60 digits may time out (10 s Discord limit). The current implementation does not include GNFS or QS.
§ 07

Transcendental Functions

Implemented via CORDIC (COordinate Rotation DIgital Computer) vectoring and rotation modes, extended with argument-reduction and polynomial correction for full 2048-bit accuracy. ln and exp use a combination of CORDIC and an AGM (arithmetic-geometric mean) series.

/calc ln x
Natural logarithm
▶
Natural logarithm via AGM series. Argument reduction maps x to the interval [1, 2) before the series is applied.
example
you ▸ /calc ln 2 bot ▸ 0.69314718055994530941723212145817656807550013436025...
/calc exp x
Exponential eˣ
▶
Computes eˣ via range reduction and a Taylor series with 2048-bit intermediate products.
example
you ▸ /calc exp 1 bot ▸ 2.71828182845904523536028747135266249775724709369995...
/calc sin x | /calc cos x | /calc tan x
Trigonometric functions
▶
Argument is taken in radians. Payne-Hanek argument reduction handles very large arguments exactly (no catastrophic cancellation). CORDIC rotation produces sin and cos simultaneously; tan is derived as their ratio. Also available: asin, acos, atan, atan2, sinh, cosh, tanh.
example
you ▸ /calc sin 3.14159265358979323846 bot ▸ 2.06761537356896e-20 (near zero, as expected)
§ 08

Gamma / Factorial

/calc gamma z
Γ(z) for real / complex z
▶
Lanczos approximation with Spouge correction extended to 2048-bit. Handles non-integer arguments including negative reals (via reflection formula). Use /calc fact n for integer factorial shorthand; n! is evaluated as Γ(n+1).
example
you ▸ /calc gamma 0.5 bot ▸ 1.77245385090552 (= √π) you ▸ /calc fact 100 bot ▸ 93326215443944152681699238856266700490715968264381621468592963895217...
§ 09

Riemann Zeta

/calc zeta s
ζ(s) Riemann Zeta function
▶
Computes ζ(s) for real s > 1 using the Euler–Maclaurin formula with Bernoulli number acceleration. For even positive integers, the closed form π²ⁿ·B₂ₙ/… is used for maximal accuracy. The functional equation extends the domain to s < 0.
example
you ▸ /calc zeta 2 bot ▸ 1.64493406684822643647241516664602518921894990120680... (= π²/6) you ▸ /calc zeta 3 bot ▸ 1.20205690315959428539973816151144999076498629234049... (Apéry's constant)
§ 10

Base Conversion

/calc base n from to
Arbitrary base conversion
▶
Converts integer n from base from to base to. Supported range: bases 2–62 (digits 0–9, A–Z, a–z). Fractional parts are converted to the target base to the available precision.
example
you ▸ /calc base 255 10 16 bot ▸ FF you ▸ /calc base DEADBEEF 16 2 bot ▸ 11011110101011011011111011101111
§ 11

Balanced Ternary

Balanced ternary uses digits {−1, 0, +1} (displayed as T, 0, 1 respectively). Full arithmetic is supported natively — numbers need not be converted to binary internally.

/calc bt_add a b | bt_sub | bt_mul | bt_div
Balanced ternary arithmetic
▶
All four arithmetic operations on balanced ternary numbers. Input and output use the T/0/1 alphabet. Use bt_from and bt_to to convert to/from decimal.
example
you ▸ /calc bt_from 10 # decimal 10 → balanced ternary bot ▸ 101T # 9 + 3 + 0 − 1 = 11? No: 1·9+0·3+1·1+T·1 = 9+0+1−1=9... let's check you ▸ /calc bt_add 1T1 10T # 5 + 6 bot ▸ 1TT0 (= 11)
§ 12

Polynomial Arithmetic

Polynomials are specified as coefficient arrays in descending degree order. For example, "[1,0,-1]" represents x² − 1. All coefficients are 2048-bit values.

/calc poly_add "p" "q" | poly_mul | poly_div | poly_gcd
Polynomial operations
▶
poly_add / poly_sub — Coefficient-wise addition/subtraction.
poly_mul — Schoolbook O(n²) polynomial multiplication.
poly_div — Returns quotient and remainder pair (polynomial long division).
poly_gcd — GCD of two polynomials via subresultant PRS (pseudo-remainder sequence), avoiding coefficient explosion.
example
you ▸ /calc poly_mul "[1,-1]" "[1,1]" # (x−1)(x+1) bot ▸ [1, 0, -1] # x² − 1 you ▸ /calc poly_gcd "[1,0,-1]" "[1,-1]" # gcd(x²−1, x−1) bot ▸ [1, -1] # x − 1
§ 13

Symbolic Operations

/calc sum "expr" var lo hi
Symbolic / numeric summation
▶
Evaluates Σ expr(var) for var = lo..hi. If hi is inf, attempts symbolic closed-form recognition first (geometric series, Basel-type), then falls back to accelerated summation (Euler–Maclaurin, Aitken Δ²).
example
you ▸ /calc sum "1/n^2" n 1 inf bot ▸ 1.64493406684822643... (= π²/6, recognised symbolically)
/calc prod "expr" var lo hi
Symbolic product Π
▶
Evaluates ∏ expr(var) for var = lo..hi. Infinite products are evaluated by converting to a sum of logarithms and applying summation acceleration.
example
you ▸ /calc prod "n/(n+1)" n 1 100 bot ▸ 0.0099009900990099... (= 1/101, telescoping)

§ 15

Eigenvalues / Eigenvectors

/calc eigenval "A" | /calc eigenvec "A"
Spectral decomposition
▶
Computes eigenvalues via the QR algorithm with implicit double shifts (Francis shifts). eigenvec additionally returns the corresponding eigenvectors as column vectors. Complex eigenvalues appear as a+bi pairs.
example
you ▸ /calc eigenval "[[4,1],[2,3]]" bot ▸ λ₁ = 5.0 λ₂ = 2.0

§ 17

Linear System Solving

/calc solve "A" "b"
Solve Ax = b
▶
Solves the linear system Ax = b for x using LU factorisation with partial pivoting. If the system is underdetermined, returns the least-norm solution via QR. If overdetermined, returns the least-squares solution.
example
you ▸ /calc solve "[[2,1],[-1,3]]" "[5,0]" bot ▸ x = [1.4285714285714..., 0.7142857142857...] (= [10/7, 5/7])
§ 18

Symbolic Differentiation

/calc diff "expr" var [order]
Symbolic derivative
▶
Parses expr into an expression tree and applies symbolic differentiation rules (product rule, chain rule, quotient rule). Supports: polynomials, sin, cos, tan, ln, exp, pow. The optional order parameter specifies the derivative order (default 1).
example
you ▸ /calc diff "x^3 * sin(x)" x bot ▸ 3*x^2*sin(x) + x^3*cos(x) you ▸ /calc diff "ln(x^2 + 1)" x 2 bot ▸ (2*(x^2+1) - 4*x^2) / (x^2+1)^2 → simplified: (2-2x²)/(x²+1)²
§ 19

Numerical Integration

/calc integrate "expr" var a b [method]
Definite integral
▶
Computes ∫ₐᵇ expr dvar. Available methods:

gauss (default) — Gauss–Legendre quadrature with adaptive node count up to 2048 points.
romberg — Romberg integration (Richardson extrapolation on the trapezoidal rule), useful for smooth periodic integrands.
Improper integrals with a=-inf or b=inf are handled via a variable substitution.
example
you ▸ /calc integrate "sin(x)/x" x 0 inf bot ▸ 1.57079632679489661923132169163975... (= π/2) you ▸ /calc integrate "exp(-x^2)" x -inf inf gauss bot ▸ 1.77245385090552 (= √π)
§ 20

Root Finding

/calc root "expr" var x0 [method] [x1]
Find f(x) = 0
▶
Finds a root of expr = 0 near the starting point(s). Methods:

newton (default) — Newton–Raphson with symbolic derivative from the expression tree. Quadratic convergence. Requires x0 near the root.
bisect — Bisection on [x0, x1]. Guaranteed linear convergence if f(x0) and f(x1) have opposite signs.
brent — Brent's method (bisection + secant + inverse quadratic). Recommended for robustness; requires bracket [x0, x1].
example
you ▸ /calc root "x^3 - 2" x 1 newton bot ▸ 1.25992104989487316476721060727822835057... (= ∛2) you ▸ /calc root "cos(x) - x" x 0 1 brent bot ▸ 0.73908513321516064165531208767387340401... (Dottie number)
§ 21

ODE Solver — Runge-Kutta RK4

/calc rk4 "f(t,y)" t0 y0 t1 steps
Solve dy/dt = f(t,y)
▶
Classic 4th-order Runge–Kutta with fixed step size h = (t1 − t0) / steps. Returns the value of y at t1, plus a formatted table of intermediate values if steps ≤ 20. For systems, y0 may be a bracketed vector.
example
you ▸ /calc rk4 "y" 0 1 1 4 # dy/dt = y, y(0)=1 → should give e t=0.00 y=1.000000000000 t=0.25 y=1.284025416687 t=0.50 y=1.648721270700 t=0.75 y=2.117000016612 bot ▸ t=1.00 y=2.718281828460
ℹ Adaptive step-size control (RK4/5 Dormand–Prince) is planned for a future release. For stiff systems, consider reformulating or using a small fixed step size.
